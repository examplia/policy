01. INTRODUCTION

This Privacy Policy describes how Buck ("we," "us," or "the Bot") collects, uses, and protects your information when you interact with our Discord bot services. We are committed to protecting your privacy and ensuring transparency about our data practices.

02. INFORMATION WE COLLECT

2.1 Automatically Collected User Information

  • Discord User ID: Unique identifier assigned by Discord
  • Username and Display Name: Your current Discord username and server-specific display name
  • Account Information: Account creation date, avatar URL, discriminator, bot status
  • Server Membership Data: Join/leave timestamps, server roles, permissions, nickname
  • Premium Status: Nitro subscription status, server boost information
  • Communication Status: Timeout status, pending verification status

2.2 Enhanced Activity Monitoring

  • Message Analytics: Message count, content analysis (spam detection, link detection)
  • Content Patterns: Caps ratio, mention frequency, invite link usage
  • Voice Activity: Voice channel join/leave times, total voice time duration
  • Presence Data: Online status across devices (mobile, desktop, web)
  • Activity Information: Current Discord activity, custom status messages
  • Reaction Tracking: Message reactions and interaction patterns

2.3 Security and Behavioral Analysis

  • Suspicious Activity Scoring: Algorithmic risk assessment based on behavior patterns
  • Cross-Server Analysis: Activity patterns across multiple monitored servers
  • Join/Leave Patterns: Rapid joining, server hopping detection, stay duration
  • Message Content Analysis: Spam detection, suspicious link identification
  • Security Events: Flagged activities, warning triggers, moderation actions
  • Behavioral Patterns: Communication frequency, activity timing, interaction styles

2.4 Administrative and Moderation Data

  • Moderation Actions: Warnings, timeouts, kicks, bans, appeals
  • Flag Reports: Administrator-generated security flags and notes
  • Risk Assessments: Automated and manual security evaluations
  • Historical Records: Complete activity history for security analysis

2.5 Technical and Metadata

  • Connection Information: Client type, connection quality indicators
  • Timezone and Language: Inferred timezone offset, language preferences
  • Server Context: Verification level, server-specific settings
  • Hash Data: Content hashes for duplicate detection (not original content)

2.6 Information We Do NOT Collect

  • Full message content (only analysis patterns and hashes)
  • Private messages or direct message content
  • Voice chat recordings or transcripts
  • Financial or payment information
  • Precise location data beyond server membership
  • Personal information not publicly available on Discord

03. HOW WE USE YOUR INFORMATION

3.1 Security and Safety Purposes

  • Threat Detection: Identify potential security risks, spam, and malicious behavior
  • Cross-Server Analysis: Track suspicious activity patterns across multiple communities
  • Risk Scoring: Generate automated risk assessments based on behavioral patterns
  • Fraud Prevention: Detect account manipulation, ban evasion, and coordinated attacks
  • Community Protection: Prevent harassment, doxxing, and other harmful activities

3.2 Advanced Analytics and Monitoring

  • Behavioral Pattern Analysis: Identify unusual activity patterns and trends
  • Content Analysis: Detect spam, suspicious links, and policy violations
  • Activity Correlation: Connect related activities across time and servers
  • Predictive Modeling: Anticipate potential security threats before they escalate
  • Network Analysis: Identify coordinated groups and suspicious networks

3.3 Moderation and Administrative Support

  • Decision Support: Provide comprehensive user profiles for moderation decisions
  • Historical Context: Maintain complete activity records for informed judgments
  • Automated Flagging: Alert administrators to high-risk users and activities
  • Evidence Collection: Compile detailed reports for serious violations
  • Appeal Processing: Provide data for reviewing moderation decisions

3.4 System Optimization and Research

  • Algorithm Improvement: Enhance detection accuracy and reduce false positives
  • Performance Analytics: Monitor system effectiveness and user impact
  • Security Research: Study emerging threats and attack patterns
  • Compliance Monitoring: Ensure adherence to platform policies and regulations

04. INFORMATION SHARING AND DISCLOSURE

4.1 Authorized Sharing

  • Server Administrators: Information is shared with administrators of monitored servers
  • Cross-Server Reports: User activity may be reported across different monitored communities
  • Security Alerts: High-risk user notifications sent to relevant server staff

4.2 We Do NOT Share Information With

  • Third-party companies for marketing purposes
  • Data brokers or advertising networks
  • Unauthorized individuals or organizations
  • Government agencies (except when legally required)

4.3 Legal Disclosure

We may disclose information when:

  • Required by law or legal process
  • Necessary to protect our rights or safety
  • Requested by Discord for Terms of Service violations
  • Required to prevent fraud or security threats

05. DATA RETENTION

5.1 Retention Periods

  • Active User Data: Retained while you remain in monitored servers plus 90 days
  • Behavioral Analysis Data: Maintained for up to 3 years for pattern recognition
  • Security Event Logs: Retained for up to 5 years for investigation purposes
  • Flagged User Records: Retained indefinitely for security and safety purposes
  • Cross-Server Analysis: Maintained for up to 2 years for trend analysis
  • Message Analysis Patterns: Retained for 1 year (content hashes only)
  • Risk Assessment Data: Maintained for 3 years for algorithm improvement

5.2 Extended Retention for Security

  • High-Risk Users: Data retained longer based on threat assessment
  • Investigation Records: Maintained until case resolution plus 2 years
  • Ban Evasion Data: Retained indefinitely to prevent circumvention
  • Coordinated Attack Data: Preserved for network security analysis

5.3 Data Deletion and Purging

  • Automated purging occurs when retention periods expire
  • Users may request data deletion (subject to security and legal considerations)
  • Critical security data may be retained longer if required for ongoing investigations
  • Anonymized aggregate data may be retained indefinitely for research purposes

06. DATA SECURITY

6.1 Technical Security Measures

  • Advanced Encryption: AES-256 encryption for data at rest, TLS 1.3 for data in transit
  • Database Security: Multi-layer database protection with query monitoring
  • Access Controls: Role-based access with multi-factor authentication
  • Network Security: Firewalls, intrusion detection, and DDoS protection
  • Data Integrity: Checksums and hash verification for all stored data

6.2 Operational Security

  • Continuous Monitoring: 24/7 security monitoring and threat detection
  • Incident Response: Rapid response protocols for security breaches
  • Regular Audits: Monthly security assessments and penetration testing
  • Backup Security: Encrypted, geographically distributed backups
  • Vulnerability Management: Automated scanning and patch management

6.3 Data Access and Handling

  • Principle of Least Privilege: Minimal necessary access for each role
  • Comprehensive Logging: All data access logged with full audit trails
  • Regular Access Reviews: Quarterly review and validation of permissions
  • Secure Development: Security-first coding practices and code reviews
  • Data Anonymization: Personal identifiers removed from research datasets

6.4 Compliance and Standards

  • Industry Standards: Adherence to SOC 2, ISO 27001 frameworks
  • Privacy Regulations: GDPR, CCPA, and other applicable privacy laws
  • Discord ToS Compliance: Full adherence to Discord's Terms of Service
  • Regular Certification: Annual third-party security certifications
